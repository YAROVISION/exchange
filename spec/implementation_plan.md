# План впровадження: Автокорекція та Донавчання у Web UI

Цей план описує інтеграцію двох нових інтелектуальних функцій у веб-інтерфейс Kronos:
1. **Автокорекція систематичної похибки (Bias/Residual Correction)**: динамічне коригування прогнозної кривої на основі аналізу останніх помилок моделі на історичних даних.
2. **Панель донавчання моделі (Fine-Tuning Control)**: можливість запускати донавчання моделі на актуальних даних Binance безпосередньо з веб-інтерфейсу у фоновому режимі з відображенням логів у реальному часі.

---

## User Review Required

> [!IMPORTANT]
> **Час навчання на CPU:** 
> Процес донавчання (`train_sequential.py`) виконується на процесорі (CPU) або MPS. Для 5 епох на CPU це триватиме приблизно 30–60 секунд. Процес буде запущено асинхронно, щоб інтерфейс не блокувався.
> 
> **Перезавантаження моделі:**
> Після завершення донавчання веб-сервер автоматично перезавантажить модель `btc-daily-finetuned` з диска, щоб нові прогнози відразу враховували оновлені ваги.

- **Реалізація автокорекції в `/api/predict`**:
  - Приймати параметр `bias_correction` (boolean) з фронтенду.
  - Якщо `true` і довжина вхідних даних достатня:
    1. Відрізати останні $N=10$ точок від вхідної історії.
    2. Зробити внутрішній прогноз на ці 10 точок.

    4. Розрахувати середню різницю (фактична ціна мінус прогнозована) окремо для колонок `open`, `high`, `low`, `close`.
    5. Зсунути фінальний прогноз на ці значення (зробити зсув вгору або вниз).
    6. Застосувати фізичні обмеження (High >= Max(Open, Close), Low <= Min(Open, Close), ціни > 0).
    7. Повернути коефіцієнти зміщення у відповіді JSON для відображення в UI.

- **Додавання фонового тренування**:
  - Створити глобальний об'єкт стану донавчання:
    ```python
    finetune_state = {
        'is_running': False,
        'pid': None,
        'start_time': None,
        'epochs': 0,
        'progress_percent': 0,
        'current_epoch': 0,
        'log_file': None,
        'error': None
    }
    ```
  - Створити ендпоінт `@app.route('/api/finetune/start', methods=['POST'])`:
    - Перевіряти чи тренування вже не запущено.
    - Визначати кількість епох (параметр `epochs`, за замовчуванням 5).
    - Отримувати поточний шлях до оновленого файлу даних.
    - Запускати `train_sequential.py` через `subprocess.Popen` з аргументами `--skip-tokenizer --basemodel-epochs <epochs> --data-path <data_path> --device cpu`.
    - Перенаправляти `stdout` та `stderr` у файл логів `finetune_web_run.log`.
  - Створити ендпоінт `@app.route('/api/finetune/status', methods=['GET'])`:
    - Перевіряти статус процесу.
    - Зчитувати лог-файл, шукати рядки виду `[Epoch X/Y]` для розрахунку прогресу.
    - Повертати поточний відсоток прогресу, лог (останні 50 рядків) та статус.
    - Якщо процес завершився успішно — автоматично викликати перезавантаження моделі в пам'ять.
  - Створити ендпоінт `@app.route('/api/finetune/stop', methods=['POST'])`:
    - Вбивати процес за допомогою `process.terminate()`.

## Verification Plan

### Automated/Manual Tests

1. **Тест автокорекції (Bias Correction)**:
   - Зайти в інтерфейс, обрати модель `BTC Daily`, увімкнути чекбокс `Bias Correction`.
   - Запустити прогнозування на реальне майбутнє.
   - Перевірити в консолі сервера лог про розраховані зміщення.
   - Переконатися, що на графіку крива прогнозу плавно стикується з останньою точкою історії без розривів.
   - Перевірити прогнозування з вимкненим чекбоксом та порівняти візуальну різницю.

2. **Тест донавчання моделі (Fine-tuning)**:
   - Перейти на вкладку «Донавчання».
   - Вказати 3 епохи та натиснути «Запустити донавчання».
   - Переконатися, що з'явився прогрес-бар, а в консолі-терміналі почали з'являтися логи тренування.
   - Натиснути «Зупинити процес» під час навчання і перевірити, що процес коректно завершився і кнопка знову стала активною.
   - Запустити знову, дочекатися завершення 100%. Перевірити повідомлення про успішне перезавантаження моделі в пам'ять.
