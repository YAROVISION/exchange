# Інструкція з використання проекту Kronos

Цей проект — **Kronos: A Foundation Model for the Language of Financial Markets** — є базовою нейромережевою моделлю (foundation model) для аналізу та прогнозування фінансових свічок (K-lines / свічкових графіків). Модель була попередньо навчена на даних з понад 45 глобальних фінансових бірж.

Вона може прогнозувати ціни відкриття (`open`), закриття (`close`), максимуми (`high`), мінімуми (`low`), а також об'єми торгів (`volume`) для будь-яких активів (акцій, криптовалют тощо).

---

### 📂 Основна структура проекту

*   [model/](file:///Users/kostantinkrivula/Desktop/Kronos-master/model) — ядро моделі (реалізація архітектури Kronos, токенізатора та класу `KronosPredictor` для здійснення прогнозів).
*   [webui/](file:///Users/kostantinkrivula/Desktop/Kronos-master/webui) — локальний веб-інтерфейс на Flask для інтерактивного візуального прогнозування та порівняння з реальними даними.
*   [examples/](file:///Users/kostantinkrivula/Desktop/Kronos-master/examples) — готові скрипти для швидкого запуску CLI-прогнозів, графічного Tkinter-додатку та завантаження фінансових даних.
*   [finetune/](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune) — інструменти для тонкого навчання (fine-tuning) моделі на ваших власних даних (наприклад, акцій вашого ринку).

### 🚀 Інструкція: Як користуватись проектом (крок за кроком)

#### Крок 1. Встановлення залежностей
Перед початком запуску переконайтеся, що у вас встановлено Python 3.10 або новішої версії. Інсталюйте необхідні бібліотеки:
```bash
python3 -m pip install -r requirements.txt
```
> **Примітка:** Якщо ви плануєте робити fine-tuning, вам також знадобиться бібліотека `pyqlib` для обробки даних:
> ```bash
> python3 -m pip install pyqlib
> ```

#### Крок 2. Підготовка демонстраційних даних
Для вашої зручності створено папку `data` в корені проекту та скопійовано туди приклад реальних фінансових даних акцій Alibaba Group (`HK_ali_09988`). Тепер ви можете запускати приклади без додаткової підготовки файлів.

#### Крок 3. Запуск веб-інтерфейсу (Web UI)
Це найпростіший і найбільш наочний спосіб взаємодії з моделлю:
1. Перейдіть до папки `webui`:
   ```bash
   cd webui
   ```
2. Запустіть Flask додаток:
   ```bash
   python3 run.py
   ```
3. Відкрийте у браузері адресу: **http://localhost:7070**
4. **У веб-інтерфейсі ви зможете:**
   * Завантажити одну з моделей Kronos (`Kronos-mini`, `Kronos-small` або `Kronos-base`). Завантаження вагів з Hugging Face розпочнеться автоматично.
   * Вибрати пристрій для розрахунків (наприклад, `cpu` чи `mps` для прискорення на комп'ютерах Mac з процесорами Apple Silicon).
   * Вибрати часовий проміжок за допомогою слайдера.
   * Натиснути кнопку прогнозування та побачити інтерактивний графік прогнозу порівняно з реальною історією.

#### Крок 4. Запуск скриптів-прикладів (CLI та Desktop GUI)
У папці [examples/](file:///Users/kostantinkrivula/Desktop/Kronos-master/examples) зібрано готові демонстраційні файли:
*   **Базовий приклад передбачення з побудовою графіку:**
    ```bash
    python3 examples/prediction_example.py
    ```
*   **Запуск настільного графічного додатку (Tkinter GUI):**
    ```bash
    python3 examples/prediction_new_GUI.py
    ```
    *(Цей скрипт автоматично використовує API `akshare` для завантаження свіжих фінансових даних з бірж у реальному часі)*.

#### Крок 5. Тонке навчання моделі (Fine-tuning)
Якщо ви хочете адаптувати Kronos до вашого специфічного набору фінансових даних:
1. Задайте конфігурацію шляхів і гіперпараметрів у [finetune/config.py](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune/config.py).
2. Запустіть попередню обробку та очищення даних:
   ```bash
   python3 finetune/qlib_data_preprocess.py
   ```
3. Запустіть адаптацію токенізатора:
   ```bash
   torchrun --standalone --nproc_per_node=NUM_GPUS finetune/train_tokenizer.py
   ```
4. Запустіть донавчання моделі-передбачувача:
   ```bash
   torchrun --standalone --nproc_per_node=NUM_GPUS finetune/train_predictor.py
   ```
5. Проведіть бектестинг вашої стратегії:
   ```bash
    python3 finetune/qlib_test.py
    ```

### ⏸️ Зупинка та відновлення процесу навчання (на прикладі 20 епохи)

Якщо вам потрібно зупинити навчання на 20-й епосі та записати результати з можливістю подальшого відновлення процесу, скористайтеся одним із наступних методів.

#### Метод 1. Призупинення процесу в пам'яті (без втрати стану)
Це найпростіший спосіб зробити тимчасову паузу, якщо ви не плануєте перезавантажувати комп'ютер. Він зберігає весь стан навчання (включаючи стан оптимізатора та планувальника швидкості навчання) в оперативній пам'яті.

1. **Як поставити на паузу (після 20 епохи)**:
   Дочекайтеся, поки в терміналі з'явиться рядок про завершення 20 епохи (наприклад, `--- Epoch 20/80 Summary ---`), і натисніть комбінацію клавіш у вашому терміналі:
   `Ctrl + Z`
   Процес навчання буде призупинено, а в терміналі з'явиться повідомлення `[1]  + suspended ...`.
2. **Як відновити навчання**:
   У тому самому вікні термінала введіть команду:
   ```bash
   fg
   ```
   Навчання продовжиться з наступної епохи (21 епоха).

#### Метод 2. Повна зупинка (Ctrl + C) та «теплий старт» (Warm-Start)
Якщо вам потрібно повністю вимкнути комп'ютер чи закрити термінал, ви можете зробити повну зупинку, а потім продовжити навчання з останніх збережених ваг.

Скрипт навчання автоматично зберігає найкращу модель на диск щоразу, коли покращується метрика втрат (validation loss). Для токенізатора найкращі ваги зберігаються у папку:
`/Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/finetuned/btc_daily_finetuned/tokenizer/best_model`

1. **Як зупинити процес**:
   Дочекайтеся завершення 20-ї епохи і натисніть:
   `Ctrl + C`
   Процес завершиться. Найкращі ваги, отримані до 20 епохи включно, вже записані на вашому диску.
2. **Як відновити навчання токенізатора з 20 епохи**:
   * Відкрийте ваш файл конфігурації (наприклад, [config_btc_daily.yaml](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/configs/config_btc_daily.yaml)).
   * Змініть шлях `pretrained_tokenizer` у секції `model_paths` так, щоб він вказував на ваші збережені ваги:
     ```yaml
     model_paths:
       pretrained_tokenizer: "/Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/finetuned/btc_daily_finetuned/tokenizer/best_model"
     ```
   * Зменшіть кількість епох токенізатора на кількість уже пройдених (наприклад, з `80` до `60` епох):
     ```yaml
     training:
       tokenizer_epochs: 60
     ```
   * Запустіть навчання знову. Модель завантажить ваги 20-ї епохи як базові та продовжить навчання.
3. **Як перейти одразу до навчання базової моделі (якщо токенізатор повністю готовий)**:
   Якщо 20 епох для токенізатора було достатньо і ви хочете почати навчання самої прогнозуючої моделі (Basemodel), запустіть скрипт із прапорцем пропуску токенізатора:

#### Метод 3. Програмне збереження повного стану (Чекпоінтів)
Для того щоб автоматично зберігати повний стан навчання (номер епохи, ваги моделі, стан оптимізатора та планувальника) кожні N епох, необхідно модифікувати код навчання.

##### Крок 1. Зміна коду навчання токенізатора
У файлі [finetune_tokenizer.py](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/finetune_tokenizer.py) всередині функції `train_tokenizer` додайте код збереження чекпоінту в кінці епохи:
```python
# Вставити всередину циклу for epoch in range(config.tokenizer_epochs):
if (epoch + 1) % 20 == 0:
    checkpoint = {
        'epoch': epoch,
        'model_state_dict': model.state_dict(),
        'optimizer_state_dict': optimizer.state_dict(),
        'scheduler_state_dict': scheduler.state_dict(),
        'best_val_loss': best_val_loss,
    }
    torch.save(checkpoint, os.path.join(save_dir, f'checkpoint_epoch_{epoch+1}.pt'))
    logger.info(f"Збережено повний чекпоінт на епосі {epoch+1}")
```

##### Крок 2. Зміна коду завантаження чекпоінту
Перед початком циклу навчання додайте перевірку наявності чекпоінту:
```python
# Приклад завантаження перед циклом навчання
checkpoint_path = os.path.join(save_dir, 'checkpoint_epoch_20.pt')
if os.path.exists(checkpoint_path):
    checkpoint = torch.load(checkpoint_path, map_location=device)
    model.load_state_dict(checkpoint['model_state_dict'])
    optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
    scheduler.load_state_dict(checkpoint['scheduler_state_dict'])
    start_epoch = checkpoint['epoch'] + 1
    best_val_loss = checkpoint['best_val_loss']
    logger.info(f"Успішно відновлено навчання з епохи {start_epoch}")
```
Аналогічний код можна вбудувати у [finetune_base_model.py](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/finetune_base_model.py) для фази навчання прогнозуючої моделі.

#### ⚙️ Корекція систематичної похибки (Bias Correction)

Модель Kronos, як і будь-яка нейромережа, має систематичну похибку — тенденцію стабільно переоцінювати або недооцінювати ціни у певному напрямку. Функція **Автокорекція** усуває цей зсув автоматично.

**Де знаходиться:** вкладка «🔮 Прогнозування» → чекбокс **«⚙️ Корекція систематичної похибки (Bias Correction)»** (увімкнений за замовчуванням).

**Як це працює:**

Перед побудовою фінального прогнозу система виконує мінірегресію (backtest):
1. Бере останні **10 точок** з наявної історії, приховує їх від моделі.
2. Просить модель передбачити ці 10 прихованих точок.
3. Порівнює передбачення з реальними значеннями — обчислює зміщення (offset) для кожного типу ціни (`open`, `high`, `low`, `close`).
4. Застосовує **зважене усереднення** (вищі ваги для свіжіших точок), щоб усунути вплив застарілих даних.
5. Додає розраховане зміщення до всього прогнозного ряду перед поверненням результату.

Після прогнозу в інтерфейсі відображається блок «Застосована Bias Correction» із конкретними значеннями зсуву для кожного цінового параметра (наприклад: `Open: +143.22$`, `Close: -87.45$`).

**Обмеження:** функція активується лише якщо у вхідних даних є не менше ніж **300 рядків**.

#### 🔄 Автоматичне оновлення даних BTC (Binance API)

Якщо у розділі «Оберіть файл даних» обраний файл із денними даними біткоїна (`btc_history_daily.csv`), то перед кожним запуском прогнозування програма **автоматично перевіряє** наявність нових свічок на біржі Binance.

**Логіка роботи:**

1. Визначається остання дата у файлі `btc_history_daily.csv`.
2. Якщо поточна дата UTC більша за останню дату у файлі — надсилається запит до Binance Klines API (`BTCUSDT`, `1d`) для завантаження всіх відсутніх денних свічок.
3. Нові дані об'єднуються з існуючою історією та зберігаються в окремий файл `data/btc_history_daily_latest.csv` — щоб оригінальний файл залишався незмінним.
4. Прогнозування виконується на основі оновленого файлу.
5. В інтерфейсі відображається повідомлення про те, що дані були оновлені, і назва нового файлу автоматично підставляється в селектор.

Якщо дані вже актуальні (остання дата у файлі відповідає поточній) — оновлення пропускається, і прогнозування запускається відразу.

- [`data/btc_history_daily.csv`](file:///Users/kostantinkrivula/Desktop/Kronos-master/data/btc_history_daily.csv) — вхідний базовий файл.

#### 🎓 Донавчання моделі через веб-інтерфейс (Fine-tuning Tab)

Вкладка **«⚙️ Донавчання»** дозволяє виконати додаткове навчання моделі `BTC Daily (Донавчена)` безпосередньо з браузера, без запуску командного рядка.

**Як запустити:**

3. Натисніть кнопку **«▶ Запустити донавчання»**.

**Що відбувається під капотом:**

- Бекенд (`app.py`) запускає скрипт [`finetune_csv/train_sequential.py`](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/train_sequential.py) як **фоновий процес** (subprocess) зі стандартними параметрами конфігурації [`configs/config_btc_daily.yaml`](file:///Users/kostantinkrivula/Desktop/Kronos-master/finetune_csv/configs/config_btc_daily.yaml).

- Прапорець `--skip-tokenizer` дозволяє пропустити фазу навчання токенізатора та відразу перейти до донавчання прогнозуючої базової моделі.
- Лог виконання зберігається у `webui/finetune_web_run.log`.

Після запуску інтерфейс кожні 3 секунди автоматично опитує ендпоінт `/api/finetune/status` і оновлює:
- **Прогрес-бар** із відсотком виконання та номером поточної епохи.

#### 🏦 Інтерактивний симулятор торгівлі (Вкладка «Біржа»)

Вкладка **«🏦 Біржа»** на панелі керування додає інтерактивну мінігру-симулятор, яка дозволяє спробувати себе в ролі трейдера та імітує процес купівлі й продажу біткоїна за реальними котируваннями.

**Основні можливості та правила гри:**

1. **Початковий баланс:**
   Кожна нова гра починається з балансом **$100.00 USD** та **0.00000000 BTC**. Ця сума зберігається локально у браузері (`localStorage`), тому ваш прогрес не зникне при оновленні сторінки.

2. **Поточний курс BTC:**
   Ціна біткоїна у грі оновлюється кожні **5 секунд** автоматично через прямі запити до публічного API біржі Binance. Якщо інтернет-з'єднання з серверами Binance відсутнє, гра автоматично перемикається на отримання останньої зафіксованої ціни з вашого локального файлу історії.

3. **Комісія за операції:**
   Для максимальної відповідності реальним торгам, при кожній операції купівлі чи продажу стягується стандартна комісія у розмірі **0,1%** від загального об'єму угоди:
   - **При купівлі:** Списується вартість BTC + 0,1% комісії (`сума_USD * 1.001`).
   - **При продажу:** Зараховується виручена сума USD за вирахуванням 0,1% комісії (`сума_USD * 0.999`).

4. **Керування ордерами (Купівля / Продаж):**
   - Виберіть режим операції за допомогою перемикача **Купівля** або **Продаж** (одночасно активний лише один режим).
   - Введіть потрібну кількість біткоїнів у текстове поле **«Кількість біткоїнів:»**. Завдяки оптимізації поля вводу, будь-які малі дробові значення (аж до `0.00000001` BTC) відображаються у нормальному вигляді та не перетворюються браузером на наукову нотацію (1e-8).
   - Ви можете використовувати кастомні кнопки **`+`** та **`-`** поруч із полем для швидкого крокового регулювання суми (зміна на `0.0001 BTC` за один клік).
   - **Динамічний еквівалент:** Під полем вводу відображається жива оцінка вартості в доларах та фінальна сума з урахуванням комісії (наприклад: *з комісією 0.1%* або *після комісії 0.1%*).

5. **Журнал активних угод (Status Log):**
   - Усі здійснені покупки BTC відображаються у блоці **«Статус операцій»**. 
   - Якщо ви купуєте біткоїни декількома частинами, гра автоматично розраховує вашу **середньозважену ціну входу (Weighted Average Price)** за формулою:
     $$\text{Курс купівлі} = \frac{(\text{BTC}_1 \times \text{Курс}_1) + (\text{BTC}_2 \times \text{Курс}_2)}{\text{BTC}_1 + \text{BTC}_2}$$
   - При продажу всієї позиції запис очищується до стану «Активні операції відсутні».

6. **Скидання результатів:**
   У будь-який момент ви можете скинути свій баланс гаманця та видалити активні операції до початкових $100.00, натиснувши кнопку **«🔄 Скинути баланс»** у самому низу вкладки.

#### 🔐 Авторизація, розмежування ролей та хмарна синхронізація (Вкладка «Вхід/Реєстрація»)

Для захисту даних користувачів, безпеки процесу навчання моделі та збереження прогресу симулятора торгівлі, у версію 2.0 було впроваджено систему авторизації на базі **Supabase Auth** та хмарну базу даних PostgreSQL.

**Основні компоненти та функції безпеки:**

1. **Реєстрація та Вхід (Supabase Auth):**
   - У лівому меню з'явилася нова вкладка **`🔐 Вхід/Реєстрація`**.
   - Користувачі можуть зареєструватись за допомогою свого email та пароля. Після реєстрації в базі даних Supabase автоматично створюється особистий віртуальний гаманець із початковим балансом **$100.00 USD**.
   - Вхід здійснюється за захищеним протоколом, Flask-сервер зберігає сесію та токен авторизації.

2. **Хмарне збереження стану (PostgreSQL):**
   - Усі дані гаманця (USD баланс, BTC баланс, середня ціна позиції) та історія угод зберігаються в хмарних таблицях `wallets` та `trades` замість локального сховища браузера (`localStorage`).
   - Це повністю вирішує проблему безпеки та запобігає маніпуляціям із балансом через панель розробника в браузері (Developer Tools Injection Protection). Усі операції проходять валідацію на Flask-бекенді.

3. **Розмежування прав та активність елементів інтерфейсу:**
   - **Неавторизовані користувачі:**
     - Усі кнопки керування угодами на вкладці «🏦 Біржа» та кнопки запуску/зупинки на вкладці «⚙️ Донавчання» є неактивними (`disabled`).
     - Відображається плашка з проханням увійти в систему.
   - **Звичайні авторизовані користувачі:**
     - Мають повний доступ до вкладки «🏦 Біржа» (кнопки торгівлі та скидання балансу стають активними).
     - Кнопки на вкладці «⚙️ Донавчання» залишаються неактивними з повідомленням про обмежений доступ.
   - **Головний адміністратор:**
     - Має повний безперешкодний доступ до всіх функцій додатку (як до біржі, так і до запуску чи зупинки донавчання моделі).

4. **Бекенд-захист (API Security):**
   - Ендпоінти донавчання `/api/finetune/start` та `/api/finetune/stop` перевіряють email авторизованого користувача. У разі спроби виклику іншим користувачем сервер миттєво відхиляє запит із кодом `403 Forbidden`.
